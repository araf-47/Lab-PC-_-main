### Gist of "Reusable Software"

Traditionally, software developers often **created new solutions for problems that had already been solved before**, which wasted time, effort, and money.

**Reusable software** means creating software components that can be used again in multiple systems instead of building them from scratch every time.

### Why Object-Oriented Programming (OOP) Helps

Object-oriented systems are:

- **Modular** → Software is divided into independent classes and objects.
    
- **Reusable** → Existing classes can be used in new projects.
    
- **Maintainable** → Changes can be made without affecting the whole system.


### Role of Inheritance

**Inheritance** allows a new class to reuse the attributes and methods of an existing class.

Example:

- Class: `Vehicle`
    
- Subclasses: `Car`, `Bus`, `Truck`


Instead of writing common features (speed, engine, start()) repeatedly, they are defined once in `Vehicle` and inherited by the subclasses.

### Benefits of Reusable Software

1. Reduces development time.
    
2. Lowers costs.
    
3. Improves reliability (reused components are already tested).
    
4. Increases productivity.
    
5. Promotes consistency across systems.


### One-line exam answer

**Reusable software refers to software components that can be used in multiple applications, reducing development time and cost. Object-oriented development supports reusability through modular design and inheritance.**
